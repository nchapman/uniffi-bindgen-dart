## Project Overview

uniffi-bindgen-dart is a Dart backend for UniFFI — it generates idiomatic Dart bindings from UDL definitions. Built in Rust, it generates Dart code that uses `dart:ffi` and `DynamicLibrary` to call into compiled Rust shared libraries (.dylib/.so/.dll).

## Build & Test Commands

```bash
# Build Rust workspace
cargo build --workspace

# Run Rust tests (unit + golden)
cargo test --workspace

# Full CI check (run before committing)
cargo fmt --check && cargo clippy --all-targets -- -D warnings && cargo test --workspace

# Build fixtures, generate bindings, and run Dart runtime tests
./scripts/test_bindings.sh

# Generate bindings from a UDL file
cargo run -- generate fixtures/simple-fns/src/simple-fns.udl --out-dir /tmp/out

# Run a single Rust test
cargo test -p ubdg_bindgen golden_simple_fns
cargo test -p ubdg_bindgen golden_coverall_demo
```

## Architecture

### Crate Layout

- **`crates/ubdg_bindgen`** — Main code generator. CLI entrypoint, config parsing, Dart output.
- **`crates/ubdg_runtime`** — Runtime support library for generated Dart bindings.
- **`crates/ubdg_testing`** — Shared test utilities for fixture and integration testing.

### Generator Pipeline

```
main.rs → cli.rs (Clap) → lib.rs::run() → dart/mod.rs::generate_bindings()
```

`dart/mod.rs` is the core generation module (~9000 lines). It resolves config from an optional TOML file (`[bindings.dart]` section), derives the library name from the UDL filename, and renders Dart output.

### Generated Output Format

The generator produces a self-contained Dart file that uses `dart:ffi` for native interop:

```dart
// Generated by uniffi-bindgen-dart. DO NOT EDIT.
library simple_fns;

import 'dart:ffi' as ffi;
import 'package:ffi/ffi.dart';

class SimpleFnsFfi {
  // FFI lookups and Dart wrappers
}
```

**Convention**: The Dart library name matches the UDL namespace in snake_case. Generated files include FFI class (`{Name}Ffi`), public wrapper functions, type codecs (`*FfiCodec`), and a configurable `DynamicLibrary` binding system.

### Native FFI Architecture

For each fixture:
1. Rust cdylib is compiled (`.dylib`/`.so`) exporting C functions
2. `uniffi-bindgen-dart generate` reads UDL and emits `{namespace}.dart`
3. Consumer opens the library: `configureDefaultBindings(libraryPath: 'libmy_crate.dylib')`

### Golden Test Pattern

Fixtures live in `fixtures/{name}/` with UDL input at `src/{name}.udl` and expected output at `expected/{name}.dart`. The test harness (`crates/ubdg_bindgen/tests/golden_generated.rs`) generates bindings to a temp dir and compares byte-for-byte against expected files. When changing generator output, regenerate expected files:

```bash
cargo run -- generate fixtures/{name}/src/{name}.udl --out-dir /tmp/regen
cp /tmp/regen/{namespace}.dart fixtures/{name}/expected/{namespace}.dart
```

### Naming Conventions

- Generated library names use snake_case (e.g., `simple_fns`)
- Exported Dart names use camelCase for functions/methods, PascalCase for types
- Crate names prefixed with `ubdg_`
- UDL filenames use kebab-case (`simple-fns.udl`)

### External Type Support

External types are declared in UDL with `[External="crate_name"]` and configured in `uniffi.toml`:
```toml
[bindings.dart]
external_packages = { other_crate = "package:other_bindings/other_bindings.dart" }
```

## Development Methodology

- **TDD-first**: Write failing tests before implementation for each UDL feature.
- **Idiomatic output**: Generated code must look like hand-written Dart, not Rust transliterations.
- **Deterministic generation**: Golden tests enforce reproducible output.
- **Linear history**: Keep commits as coherent units with descriptive messages in imperative mood.
